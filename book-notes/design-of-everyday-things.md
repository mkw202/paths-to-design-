# The Design of Everyday Things

- **Author:** Don Norman
- **Editions:** 1988 (as *The Psychology of Everyday Things*), revised
  and expanded 2013
- **Field:** Foundational design theory / cognitive ergonomics

## Thesis
Bad design isn't bad users. When people struggle, the system has failed
to communicate its capabilities (affordances), signal them (signifiers),
respond to action (feedback), or match the user's mental model of how
things should work.

## Core ideas
- **Affordances vs. signifiers.** An affordance is what an object *can*
  do; a signifier is the cue that tells the user it can. A door affords
  being pushed AND pulled — the handle is the signifier that disambiguates.
- **Mental models.** The designer has one. The user has one. The system
  has an image. These three rarely align. The system image is the only
  thing the user actually sees — make it carry the model.
- **Seven stages of action.** Goal → plan → specify → perform → perceive
  → interpret → compare. Most failures sit at one of these seven points.
- **Gulfs of execution and evaluation.** The execution gulf is the gap
  between intent and possible action. The evaluation gulf is the gap
  between system state and user understanding. Good design closes both.
- **Mapping.** Natural mappings (stove burner layout matching knob
  layout) eliminate cognitive translation.
- **Constraints.** Physical, logical, cultural, and semantic constraints
  prevent error by making wrong actions impossible or unobvious.
- **Feedback.** Every action needs visible, immediate, informative
  response. No feedback = anxiety; wrong feedback = mistrust.
- **Errors: slips vs. mistakes.** Slips are right-intent-wrong-execution
  (typos, mis-clicks). Mistakes are wrong-intent (planning errors). They
  need different design responses; both must be designed for.
- **Human-centered design.** Observe, generate ideas, prototype, test,
  iterate. Treat users as the source of requirements.

## Named frameworks & heuristics
- The **Norman door** — the canonical example of an affordance/signifier
  mismatch.
- **Seven-stage action cycle.**
- **Gulfs of execution and evaluation.**
- The **paradox of technology** — added features add capability and
  complexity, often non-linearly.

## How to apply
- When a flow feels confusing, name which gulf the user is stuck in —
  execution (don't know what's possible) or evaluation (don't know what
  happened).
- For errors: design for slips (confirm destructive actions, allow undo)
  and mistakes (clear feedback when assumption is wrong).
- Map controls to outcomes spatially when possible.
- Audit every interaction for a clear, immediate feedback signal.

## When to reach for it
A career foundation. Read once carefully, revisit every couple of years.
The vocabulary it gives you (affordance, signifier, mapping, gulfs) you
will use forever in design conversations.

## Caveats
- Long. The 2013 revision is the edition to buy — clearer prose, modern
  examples, additional human-centered-design chapter.
- Examples are physical-world heavy (doors, stoves, light switches); you
  translate to interfaces yourself.
- Light on actionable visual or interaction patterns — it's a theory
  book.
