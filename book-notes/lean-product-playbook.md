# The Lean Product Playbook

- **Author:** Dan Olsen
- **Edition:** 2015
- **Field:** Product management — zero-to-one and product/market fit

## Thesis
Problem/solution fit and then product/market fit are reachable by
iterating on a concrete six-step framework, not by following Lean
Startup as a slogan. The framework: target customer → underserved needs
→ value proposition → feature set → UX → measure → iterate.

## Core ideas
- **Product/market fit pyramid.** Bottom to top: market → underserved
  needs → value proposition → feature set → UX. Each layer must rest on
  the layer below.
- **Target customer first.** Specificity is the point. "Small business
  owners" is not a customer; "10-person dental practice managing
  paperwork" is.
- **Underserved needs over wants.** Importance + dissatisfaction is the
  pair that identifies opportunity. High-importance, high-satisfaction
  needs are already won; chase high-importance, low-satisfaction ones.
- **Value proposition before features.** A clear one-sentence value
  proposition forces hard tradeoffs; jumping to features hides them.
- **MVP definition is precise, not loose.** "Minimum" = smallest scope
  that tests the riskiest assumption; "viable" = enough to be
  evaluable. An MVP is a hypothesis test, not a half-built product.
- **Kano model.** Must-haves (table stakes), performance attributes
  (more is better), delighters (unexpected joy). Each category needs
  different design attention.
- **Importance vs. satisfaction grid.** Plot needs; the upper-left
  quadrant (important, low-satisfaction) is your opportunity.
- **Build-measure-learn with discipline.** Define what each loop is
  designed to learn before building. Don't ship and then ask what to
  measure.

## Named frameworks & heuristics
- **Product/market fit pyramid** (the signature contribution).
- **Importance-satisfaction gap analysis.**
- **Kano model** (applied; not original to Olsen).
- **Six-step lean product process.**

## How to apply
- Write the **target customer** as a specific, narrow archetype.
  "Anyone" is wrong.
- Write the **value proposition** as a single sentence the customer
  would agree with. If you can't, you haven't found one.
- Rank needs by importance × dissatisfaction; build for the
  highest-product, not the loudest customer.
- Before any build, write the assumption you're testing and what
  evidence would falsify it.

## When to reach for it
Zero-to-one product work; validating a new product line; teaching
junior PMs the actual mechanics of finding fit. Best companion to *Lean
Startup*, which is more philosophical.

## Caveats
- Prescriptive — the framework is rigid, which is its strength and its
  weakness.
- Can feel mechanical applied to mature products that already have fit.
- Light on discovery practice itself — pair with
  *Continuous Discovery Habits*.
