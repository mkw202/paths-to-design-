# Thinking with Type

- **Author:** Ellen Lupton
- **Editions:** 1st (2004), 2nd expanded (2010)
- **Field:** Typography fundamentals

## Thesis
Typography is a system of three nested concerns — letter, text, grid —
each with its own craft rules. Mastering all three (in order) is what
separates typography from "font picking."

## Core ideas

### Letter
- **Anatomy.** Cap height, x-height, ascender/descender, baseline,
  counter, stroke contrast, axis. Knowing the parts is the prerequisite
  for choosing and pairing fonts.
- **Type families and classifications.** Old style, transitional,
  modern/Didone, slab, sans (humanist / geometric / grotesque), display.
  Classification predicts feel and use.
- **Kerning and tracking.** Kerning fixes problem pairs; tracking
  adjusts overall letter-spacing. Headlines benefit from negative
  tracking; small caps and all-caps benefit from positive tracking.

### Text
- **Line length (measure).** 45–75 characters per line for body text.
  Outside that range, reading suffers.
- **Leading (line-height).** Generally 120–145% of size for body. Longer
  measure → looser leading.
- **Alignment.** Left-aligned ragged for most screen contexts. Justified
  text demands real hyphenation; without it, justified body is bad
  typography.
- **Hierarchy without size.** Weight, case, position, color, and space
  carry as much hierarchy as size — usually with more elegance.
- **Indents and paragraphs.** Indent OR space, not both. One signal
  is enough.

### Grid
- Ties type to layout (see *Grid Systems*).
- Baseline grids preserve vertical rhythm across columns.

### Voice
- **Concept-first font selection.** Pick type that fits the content's
  voice, not the designer's preference. "What does this say?" before
  "what looks nice?"
- **Readability vs. legibility.** Legibility = letters distinguishable.
  Readability = comfort of extended reading. Different goals; different
  optimizations.

## Named frameworks & heuristics
- **Letter / text / grid** three-layer model.
- **Measure of 45–75 characters.**
- **Hierarchy by weight + case + position**, not just size.

## How to apply
- Set body type measure within 45–75 chars; if it falls outside, change
  font size or column width.
- Use a single type family with multiple weights before reaching for a
  second family.
- When pairing, contrast classifications (e.g., a humanist sans with a
  transitional serif) rather than choosing two similar fonts.
- Pick the type system first; design the rest around it.

## When to reach for it
When you find yourself second-guessing every font, size, and spacing
choice. Also as a foundation before you go deeper into editorial design.

## Caveats
- Print examples dominate; for digital-only work, pair with Matthew
  Butterick's *Practical Typography* (free online).
- Less coverage of variable fonts, screen rendering, and OpenType
  features — supplement for production work.
